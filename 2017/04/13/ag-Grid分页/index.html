AG-GRID分页

ag-grid的分页可以分为客户端分页和服务端分页，客户端分页，即后台直接将所有的数据传给前台。前台对数据进行分页和排序等等操作。而服务端操作为，每次选取的数据依旧为后台提供，同时将数据交给前台的分页组件

简单的分页

要分页必备的几个属性
  • pagination: true 开启分页，系统会自动开启默认的分页
  • paginationAutoPageSize: true 每页的显示行数自适应你定义的高度
内容 说明
pagination true 开启分页，系统会自动开启默认的分页，false（默认）
paginationPageSize 100，每页多少条默认100，如果paginationAutoPageSize存在，则无效
paginationAutoPageSize true 每页的显示行数自适应你定义的高度，false(默认)

服务端分页

服务端分页是基于ag-grid的滚动模式，即鼠标下拉加载数据的一种，变成了分页加载而已，所以有些概念无法理解

设置分页必备的几个属性

  • rowModelType: ‘infinite’ 开启无穷模式，必填
  • infiniteBlockSize: 10 每次取得数据大小，和每页行数一致
  • pagination: true
  • paginationAutoPageSize: true 我这里高度只能容纳10和上面的 infiniteBlockSize 一致，这样每次加载的数据都在一页
